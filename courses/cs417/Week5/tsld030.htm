Spread Ordering (cont.)

  • Delivery consists of applying 4 ordered tests to the Message List and delivering any messages which pass the tests to the session layer

    • Any message <=“SAFE” (LTS< ARU) can be delivered and deleted
    • Then any message <= “AGREED” (LTS < My_Site_ARU) upto first SAFE message
    • Then any message <= “FIFO” for which (session_seq < Max FIFO deliverable seq) or is RELIABLE
    • Unreliable messages can be delivered anytime and might be delivered faster then identical RELIABLE messages
