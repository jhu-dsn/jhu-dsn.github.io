ISIS Characteristics

  • The algorithms are implemented at the process level. A library is linked with the application.

  • Close groups semantics.

  • Initially point to point (TCP/IP). Later: improved.

  • Heterogeneous system architecture.

  • Fail-stop failure model.

  • Services: Causal order, Total Order, Stability.

  • The Programmer recommends the group type: Peer, Client-server, Diffusion, Hierarchical.
