Design
The DNS has one entry for the web server in question, when the client attempts to resolve this name to an IP address the DNS returns the actual IP address of the proxy server. Then the client connects to the proxy server and the proxy server decides which underlying server is best suited to respond. All transactions of that session are then forwarded by the proxy to that server.
Proxied Servers Model
Advantages:
  • Dynamic load balancing is possible.
  • CPU power, disk throughput expandable (add computers).

    • Disadvantages:
  • Limited bandwidth (into proxy).
  • Proxy adds a small overhead to each connection.
  • Compilcated configuration.
  • Redundancy in the servers, but the proxy is a vertex cut

    • Comments:

    This model is extremely attractive when server CPU utilization is high, because of the ability to intelligently distribute requests. However, the proxy is attractive point for malicious hackers to focus denial of service (DOS) attacks.
