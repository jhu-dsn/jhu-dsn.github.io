HTTP/1.0 Check out the RFC
  • Single transaction per connection.
  • Allowable transactions are very basic:
  • GET
  • HEAD
  • POST
  • "Update if newer" concept implemented.

    • Motivations: The HTTP was entering the public field of focus and a specification that was strict and standardized was necessary. Few changes to the protocol actually occured in the transition from 0.9 to 1.0, as the main issue was formal specification.
