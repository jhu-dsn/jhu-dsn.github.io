Design
The DNS has multiple entries for the web server in question. At each request, the DNS will rotate the list with which it responds. This way each new connecting client will get a a different server than the previous one. This solution will distribute the load without any modifications to the topology of the server farm.
DNS Server Rotation Model
Advantages:
  • Simple configuration (a few additional DNS records).
  • Load distribution is possible.
  • Redundancy in the servers.
  • Unlimited bandwidth by adding servers.
  • CPU power, disk throughput expandable (add computers).

    • Disadvantages:
  • Load balancing is ignorant!!! (doesn't perform well when implemented)

    • Comments:

    This model is widely used on the Internet by large, high traffic sites. It allows servers to be geographically diverse. It's ease of setup and the simplicity of adding additional servers makes it more popular.
