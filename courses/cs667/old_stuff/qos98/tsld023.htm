RSVP - Operation

  • Sender sends Path messages downstream towards an RSVP receiver destination

  • Path messages are used to store path information in each node in the traffic path

  • After receiving a Path message, the RSVP receiver sends Resv messages back upstream to the sender

  • Resv messages specify the desired QoS and set up reservation state in each node in the traffic path

  • After receiving the Resv message, the sender sends its data
