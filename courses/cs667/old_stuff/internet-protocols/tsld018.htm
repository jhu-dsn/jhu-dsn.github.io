Intra-AS Routing (OSPF)

  • Once the OSPF network has been setup, it is very stable, and as such the graph is not re-calculated until there is a change in network topology.

    • When there is a change, the router that detects the change initiates a ‘flood’ which tells the rest of the OSPF network about the change. If the change involves a network that is actually in the OSPF network, then all routers must re-calculate the entire SPF graph.
