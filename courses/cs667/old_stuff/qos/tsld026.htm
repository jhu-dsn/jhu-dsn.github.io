RSVP - ReSerVation Protocol

  • Designed for use in the Internet and Mbone

  • goes beyond the Best Effort model

  • Uses a flow-spec consisting of a Traffic specification Tspec and a QoS Specification Rspec

  • Works using soft-state and periodic PATH and RESV messages

  • Admission control is if all nodes on the path accept the RESV reservation
