http://www.cs.wisc.edu/condor

  • Software system that runs on a cluster of workstations
  • Harness wasted CPU cycles
  • Supports different architectures and operating systems
  • Runs a daemon on each machine with one machine running the “master” daemon. The master daemon ensures the other daemons are running and restarts them if they crash.
  • Local daemons monitor the activity of the machine and runs a task if the machine is idle.
  • Control is return to user when user returns.
