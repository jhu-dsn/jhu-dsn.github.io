NAME

SSP_disconnect - destroy connection between application and daemon  

SYNOPSIS

#include <ssp.h>
int SSP_disconnect( mailbox mbox);  

DESCRIPTION

SSP_disconnect should be called when the application is finished with a connection to the Spread daemon. The application may have other connections still open to the daemon and may open a newq connection after disconnecting.

The mbox should be for the connection you wish to disconnect from.  

RETURN VALUES

Returns 0 on success or ILLEGAL_SESSION when the session mbox given is not a valid connection.  

BUGS

None.
