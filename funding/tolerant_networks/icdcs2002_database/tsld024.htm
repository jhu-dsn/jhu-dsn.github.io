Conclusions

  • Indeed: Group Communication tools can be employed in order to achieve Synchronous Database Replication.

  • However: Total Order is not sufficient by itself in order to guarantee strict consistency in a generic system that supports network partitions and merges, server crashes and recoveries.

  • We show: A careful, complex algorithm that overcomes the limitations of VS based approaches and that supports dynamic replica placement.

  • Furthermore: Achieved performance is up to an order of magnitude better than traditional methods.
